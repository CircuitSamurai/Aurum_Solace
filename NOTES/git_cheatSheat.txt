# Deploy Notes - Aurum_Solace

This document lists common commands and workflows used to update and deploy the Aurum backend. It assumes:

* Development is done on your **laptop**
* Deployment runs on the **mini PC**
* GitHub is the source of truth
* Python virtual environment (`venv`) is used for dependencies

---

## üß† Workflow Overview

1. **Write code on laptop**
2. **Commit & push** changes to GitHub
3. **Pull & run** latest code on the mini PC

---

## üíª Laptop: Commit & Push Changes

Use these steps every time code is updated on your laptop.

```bash
git add .
git commit -m "Describe what changed"
git push
```

### Example commit messages

```bash
git commit -m "added /coach endpoint"
git commit -m "implemented action streak tracking"
git commit -m "sqlite storage and history endpoints"
```

---

## üìù Deployment Notes

* Always **push from laptop** before pulling on mini PC
* Ensure `(venv)` is active when installing or running code
* If `git pull` fails due to conflicts, commit or stash laptop changes first
* SQLite database file (`aurum.db`) lives on the mini PC and persists between runs
